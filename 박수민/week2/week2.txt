원시타입 vs 객체타입
원시타입과 객체타입은 값이 저장되거나 복사되는 과정이 다르기 때문에 구분

원시 타입=불변값
-값 자체로써 변수에 저장되고 복사된다
-변수의 값이 수정되더라도 메모리에 한 번 저장되었던 원본 데이터가 수정되지는 않음
상수여서 불변이 아님
-메모리 값 수정X

 
객체 타입=가변값
-참조값(값에 접근할 수 있는 주소값)을 통해 변수에 저장되고 복사된다
-객체값들은 여러개의 값을 저장하고 저장하는 값의 개수가 동적이라서 별도의 메모리 공간에 보관
-메모리 상에서 수정(원본데이터수정)
-메모리 값 수정O 
-객체 타입 주의사항 
	1. 의도치 않게 값이 수정될 수 있다.
	-얕은 복사
	: 객체의 참조값을 복사함->원본 객체가 수정될 수 있어 위험
	-깊은 복사
	: 새로운 객체를 생성하면서 프로퍼티만 따로 복사 함->원본 객체가 수정될 일이 	없어 안전함

	2. 객체간의 비교는 기본적으로 참조값을 기준으로 이루어진다. 
	JSON.stringify(): 자바스크립트 내장 함수
	: 객체를 문자열로 변환하는 기능
	참조값이 아닌 프로퍼티를 기준으로 두 객체를 비교
	
	얕은 비교
	o1 == o2
	참조값을 기준으로 비교

	깊은 비교
	객체를 문자열로 변환하여 비교
	JSON.stringify 등의 내장 함수를 이용해야 함

	3. 배열과 함수도 사실 객체이다

2.13 비동기 작업 처리하기 2. promise
promise: 비동기 작업을 효율적으로 처리할 수 있도록 도와주는 자바스크립트의 내장 객체
비동기 작업들을 랩핑함

promise의 진행상태에 따른 3가지 상태
1. 대기 상태 pending
아직 작업이 완료되지 않은 상태
2. 성공 상태 fulfilled
비동기 작업이 성공적으로 마무리 된 상태
3. 실패 상태 regected
비동기 작업이 실패한 상태
대기 상태->성공상태: resolve
대기 상태->실패상태: reject

excutor함수에서 reject를 호출하게 되면 promise의 비동기 작업 실패
resolve 호출 시 비동기 작업 성공
인수로 결과값을 전달 가능

then메서드는 성공했을 때만 실행
catch메서드는 실패버전의 then

then메서드는 promise 값을 한 번 더 반환
연결하여 사용 가능
