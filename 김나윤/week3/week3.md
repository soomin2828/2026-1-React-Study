## Node.js란? 
웹 브라우저가 아닌 환경에서도 자바스크립트 코드를 실행시켜주는 자바스크립트의 런타임, 즉 자바스크립트의 실행 환경이다.

- 싷행환경(Run Time) = 구동기 :무언가를 동작시키는 기계 나 프로그램. ex) 게임기
 
 ## Node가 필요한 이유
- 자바스크립트의 히스토리
- 자바스크립트는 웹페이지 안에서 일어나는 아주 단순한 기능만을 개발하기 위해 만들어진 간단한 스크립트 언어
 ex. 버튼을 누르면 경고창뜸, 요소 색상바뀜 
 -> 생산성에만 중심을 두고 개발됨. 
 -> 웹브라우저 바깥에서도 자바스크립트를 이용해 프로그램 만들고 싶어짐
 -> Node.js등장으로 단순한 상호작용만 개발할 수 있었던 자바스크립트는 어디서든 동작할 수 있는 범용적인 언어가 됨.
 -> 자바스크립트로 웹서버(넷플릭스 등), 모바일앱(페이스북 등), 데스크톱앱(디스코드, 슬랙 등) 다 만들기 시작.

 ## Node.js 사용하기
 #### 1. 패키지 생성 및 자바스크립트를 node.js로 실행
 - 프로그래밍에서는 보통 어떠한 목적을 갖는 프로그램을 만들 때 '프로젝트'라는 단위 사용
 ex. 쇼핑몰 프로젝트

 - node.js에서는 여러개의 자바스크립트 파일로 어떠한 목적을 갖는 프로그램을 만들 때 '패키지' 라는 단위 사용
 ex. 쇼핑몰 패키지 
 -> 모든 자바스크립트 앱, 리액트 앱들도 패키지라는 단위로 만들어짐

- node.js로 자바스크립트 실행할 때 경로 확인
- 파일의 경로가 복잡해지면 패키지 스크립트 이용
- 패키지 스크립트: package.json안에 스크립트라는 항목 안에 있는 매크로들 
-> 이를 사용해 복잡한 경로에 있는 파일로 간단하게 명령 가능


#### 2. 모듈 시스템

- 모듈시스템: 모듈을 생성하고, 불러오고, 사용하는 등의 모듈을 다루는 다양한 기능을 제공하는 시스템
- 모듈: 기능별로 나뉘어진 각각의 자바스크립트 파일들 
ex. 쇼핑몰 =>
 회원관리기능 파일 user.js -> user모듈
 장바구니 기능 파일 cart.js -> cart모듈
 결제 기능 파일  payment.js -> payment모듈
 
 - JavaScript의 모듈 시스템
 1. Common JS(CJS)
 2. ES Module(ESM):CJS보다 최신식작동, 패키지 내에서 앞으로 ES모듈을 쓰겠다고 설정해줘야함 -> 패키지 설정 파일을 열고 추가적인 옵션 추가
 "type": "module"
 -> 앞으로 week3패키지는 ES모듈 시스템을 사용하겠다는 의미
 3. AMD
 4. UMD ... 
 등등

 #### 3. 라이브러리
 - 라이브러리: 프로그램을 개발할 떄 필요한 다양한 기능들을 미리 만들어 모듈화 해 놓은 것 
 - import문으로 간단히 불러오기 가능
 - npm js 사이트에 모든 라이브러리가 있음
 - node_modules폴더: 설치한 라이브러리가 실제로 저장되는 곳
 - package-lock.json: 패키지가 사용하고 있는 라이브러리들의 버전이나 정보를 package.json보다 더 정확하게 저장하는 파일
 - 'num i'라는 명령어를 터미널 창에 입력하면 모든 라이브러리를 다시 설치해줌 , 한 번 설치되어 package.json에 기록된 라이브러리들 다시 설치  -> 깃허브에 올릴 때 node_modules파일은 잘 안올림


 # <React.js>

 ## React.js란? 
 - Meta(페이스북)이 개발한 오픈소스 JavaScript라이브러리
 - 대규모 웹 서비스의 UI를 더 편하게 개발하기 위해 만들어진 기술
 - 특징
  1. 컴포넌트를 기반으로 UI를 표현
  - 컴포넌트: 화면(UI)을 구성하는 요소

  2. 화면 업데이트 구현이 쉬움
  - 이유: 리액트는 선언형 프로그래밍 방식으로 동작해서
  - 선언형 프로그래밍: 불필요한 과정은 생략하고 내가 이루고자 하는 목적만 간단히 명시하는 방법 ex. 토마토파스타 만드는 과정은 모르겠지만 토마토 파스타 하나 주세요라고 주문하는 것 (목적만 선언형으로 얘기) 
  => 업데이트를 위한 복잡한 동작을 직접 정의할 필요없이 특정 변수의 값을 바꾸는 것만으로도 화면 업데이트 가능
  - cf. 명령형 프로그래밍: 목적을 이루기 위한 모든 일련의 과정을 설명하는 방식 => 코드가 비교적 길고 복잡 

  3. 화면 업데이트가 빠르게 처리됨
  [선수지식]
  브라우저는 어떻게 동작하는가?
  HTML,CSS로 만든 페이지를 어떻게 화면에 렌더링 하는가?
  화면 업데이트는 어떻게 처리되는가?
  ![alt text](image.png)
  1단계: html -> DOM, css -> CSSOM로 변환
  2단계: 변환한 것들을 합쳐서 Render Tree로 만듦(웹페이지의 설계도,창사진)
  3단계: Layout 레이아웃 - 요소들의 배치를 잡는 작업
  4단계: Painting - 실제로 화면에 그려내는 과정

  - 여기서 자바스크립트가 DOM을 수정하면 업데이트가 발생
